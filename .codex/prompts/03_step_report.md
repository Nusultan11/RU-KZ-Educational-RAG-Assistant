# Step Report Template

Use this exact structure after each step.

Step Status:

- SUCCESS / PARTIAL / FAILED

Changed Files:

- `path/to/file`

What Changed:

- Briefly describe the change.

Why Changed:

- Explain why the change was needed.

Tests Run:

- `command`

Test Results:

- passed:
- failed:
- skipped:

Quality Check:

- formatting:
- lint:
- typing:
- unit tests:
- integration tests:
- smoke tests:
- secret check:
- random data changes check:
- large files check:

Risks:

- Remaining risks or unknowns.

Rollback Plan:

- How to revert the change.

Next Step:

- The next recommended step.

Decision:

- Can merge into feature branch: yes / no / not applicable
- Can open PR: yes / no / not applicable
- Can merge into main/master: yes / no, requires explicit user permission
