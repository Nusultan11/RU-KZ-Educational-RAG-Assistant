# Problem Formulation

## Цель проекта

Создать двуязычного образовательного RAG-ассистента для русского и казахского языков.

## Проблема

Обычная LLM может выдумывать факты и не показывать источники.

Для образовательного ассистента это риск: пользователь должен видеть, на каком контексте основан ответ.

## ML-задача

Построить retrieval + generation pipeline.

Система должна:

- искать релевантный контекст;
- генерировать ответ на его основе;
- поддерживать русский и казахский языки;
- сохранять источники;
- измерять качество метриками.

## RAG-задача

Найти релевантный контекст и ответить только на его основе.

Если контекст не найден, ассистент не должен выдумывать ответ.

## Критерии успеха

- Ответы есть на русском и казахском.
- В ответах есть источники.
- Hallucinations становятся меньше.
- Retrieval оценивается через Recall@k, MRR, nDCG.
- Pipeline можно воспроизвести через configs и отчеты.
- Preprocessing начинается только после EDA.
- RAG tuning начинается только после retrieval evaluation.
