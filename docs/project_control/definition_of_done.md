# Definition of Done

A step is done only when it produces a verifiable result and has a report.

## General Definition

A step is done when:

- scope was followed;
- required files were changed;
- unrelated files were not intentionally changed;
- checks were run or skipped with a clear reason;
- risks are documented;
- rollback plan is documented;
- next step is clear.

## Documentation-Only Done

Documentation-only work is done when:

- all requested documents exist;
- content matches the requested operating rules;
- Markdown is readable;
- no secrets are present;
- no code or business logic was changed.

## Code Done

Code work is done when:

- implementation is scoped;
- typing and architecture are consistent with project rules;
- tests are added or updated where needed;
- relevant checks pass;
- documentation is updated if behavior changed.

## ML/RAG Done

ML/RAG work is done when:

- stage prerequisites are satisfied;
- data and evaluation assumptions are documented;
- metrics are captured;
- baseline comparison exists;
- error analysis supports the next step.
